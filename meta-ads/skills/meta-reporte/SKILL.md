# Meta Reporte — Informe Ejecutivo para Clientes

Genera reportes claros, visualmente organizados y orientados a resultados de negocio. El objetivo es que el cliente entienda qué pasó, qué significa y qué sigue — sin necesitar conocimientos técnicos de Meta Ads.

## Datos necesarios para el reporte

```
Para generar el reporte necesito:

MÉTRICAS DEL PERÍODO:
□ Gasto total del período
□ Impresiones y alcance
□ Clics (enlace)
□ CTR
□ Conversiones (ventas/leads/mensajes)
□ CPA o CPL
□ ROAS (si e-commerce)
□ Frecuencia promedio

CONTEXTO:
□ Período del reporte (fechas exactas)
□ Nombre del cliente
□ Objetivo del mes
□ KPIs acordados con el cliente
□ Presupuesto ejecutado vs. presupuesto acordado
□ Principales acciones tomadas en el mes
□ Novedades del negocio del cliente (lanzamientos, cambios de precio, etc.)

COMPARATIVA (si aplica):
□ Métricas del período anterior para comparar
```

## Estructura del reporte ejecutivo

### Sección 1 — Resumen ejecutivo

El cliente lee esto primero. Máximo 3-4 oraciones. Responde:
- ¿Fue un buen mes?
- ¿Cuál fue el resultado más importante?
- ¿Hubo algo inusual o un logro destacado?

```
Plantilla:
"En [período], las campañas de Meta Ads de [cliente] generaron
[X leads/ventas] a un costo por [resultado] de [CPA/CPL],
[superando/alcanzando/por debajo de] el objetivo acordado de [KPI].
[Frase sobre contexto o logro destacado]."
```

### Sección 2 — Números del mes (dashboard de KPIs)

Tabla simple, enfocada en lo que importa al cliente:

| Métrica | Este mes | Mes anterior | Objetivo | Estado |
|---------|---------|-------------|---------|--------|
| Inversión | | | | |
| Leads/Ventas | | | | ✅/⚠️/🔴 |
| CPA/CPL | | | | |
| ROAS | | | | |
| Alcance total | | | | |

**Nota:** NO incluir CPM, frecuencia, CTR en el reporte ejecutivo del cliente (son métricas operativas para el equipo interno, no para el cliente).

### Sección 3 — ¿Qué generó los mejores resultados?

Destacar los 2-3 elementos con mejor performance:

```
Plantilla:
🏆 MEJOR CAMPAÑA: [nombre descriptivo, no técnico]
→ [resultado específico en lenguaje de negocio]

📱 MEJOR CREATIVO:
→ [descripción del anuncio + resultado]

🎯 MEJOR AUDIENCIA:
→ [descripción de quiénes respondieron mejor + resultado]
```

### Sección 4 — Insights clave

3-5 aprendizajes del mes. Escritos en lenguaje de negocio, no técnico:

```
Formato de insight:
"[Observación] → [Qué significa para el negocio]"

Ejemplos:
✓ "Las personas de 25-35 años en Bogotá convirtieron 2x más que el promedio 
   → Tiene sentido enfocar más presupuesto en este segmento el próximo mes"

✓ "Los videos cortos (< 15 segundos) generaron 40% más clics que las imágenes
   → Recomendamos producir más contenido en formato Reel"

✓ "El martes y miércoles tuvimos el CPA más bajo de la semana
   → Podemos programar mayor presupuesto en esos días"
```

### Sección 5 — ¿Qué ajustamos este mes?

Comunicar las decisiones tomadas para mantener la confianza del cliente:

```
Acciones tomadas:
□ [Acción] → [Por qué] → [Resultado]
□ [Acción] → [Por qué] → [Resultado]
```

Ejemplo:
```
□ Pausamos los 2 anuncios con más de 60 días activos
   → La audiencia los había visto demasiadas veces → Reemplazamos con nuevos creativos
□ Aumentamos el presupuesto de la campaña de retargeting un 30%
   → Estaba generando el mejor ROAS del mes → ROAS subió de 3.2x a 4.1x
```

### Sección 6 — Plan del próximo mes

Lo que viene. Claro, específico, con expectativas:

```
PRIORIDADES PARA [PRÓXIMO MES]:
1. [Acción o foco principal]
2. [Segunda prioridad]
3. [Tercera prioridad]

CREATIVOS NUEVOS NECESARIOS:
□ [Formato] — [Descripción] — Fecha límite: [fecha]
□ [Formato] — [Descripción] — Fecha límite: [fecha]

OBJETIVO KPI PRÓXIMO MES:
- [KPI principal]: [valor objetivo]
- [KPI secundario]: [valor objetivo]
```

### Sección 7 — ¿Necesitamos algo del cliente?

Lo que el equipo Sixteam necesita del cliente para el próximo mes:

```
PENDIENTES DEL CLIENTE:
□ [Material/información/decisión necesaria] — [Para qué] — [Fecha límite]
```

## Cómo comunicar resultados negativos

Cuando el mes no fue bueno, usar este framework:

### Framework AAA: Acknowledge → Analyze → Action

```
ACKNOWLEDGE (reconocer): "Este mes los resultados estuvieron por debajo de lo esperado..."
ANALYZE (analizar): "Identificamos [1-2 causas específicas y honestas]..."
ACTION (acción): "Por eso para el próximo mes vamos a [acciones concretas]..."
```

**Ejemplo:**
```
"Este mes generamos 45 leads versus el objetivo de 70. 
Identificamos dos causas: la audiencia principal mostró señales de fatiga 
(frecuencia alta) y los creativos que usamos tienen más de 2 meses activos.
Para el próximo mes ya preparamos 6 creativos nuevos con diferentes ángulos 
de mensaje, y vamos a expandir a una audiencia lookalike que no hemos probado.
El objetivo para el próximo mes es 60 leads con el nuevo material."
```

**Nunca hacer:**
- ❌ Culpar a Meta o al algoritmo sin más
- ❌ Usar excusas genéricas ("el mercado estuvo difícil")
- ❌ Ocultar los números malos
- ❌ Prometer resultados que no pueden cumplir

## Templates de mensaje para compartir el reporte

### Por WhatsApp/Mensaje de texto
```
Hola [nombre], les comparto el reporte de Meta Ads de [mes]:

📊 RESULTADOS [MES]:
✅ [X] leads/ventas generados
💰 CPA: $[monto] (objetivo: $[monto])
📈 ROAS: [X]x (si e-commerce)

[1-2 líneas del highlight más importante]

Adjunto el reporte completo. ¿Tienen disponibilidad esta semana para 
revisarlo juntos y planificar [próximo mes]?
```

### Para presentación en reunión

Estructura slide por slide:
```
Slide 1: Portada — [Cliente] | Meta Ads | [Período]
Slide 2: KPIs del mes (tabla visual simple)
Slide 3: Mejor campaña + resultado
Slide 4: Mejor creativo (screenshot) + resultado
Slide 5: Insights del mes (3 bullets máx.)
Slide 6: Plan próximo mes
Slide 7: Pendientes / Preguntas
```
